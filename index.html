What is Manual Testing?

Manual testing is to compare the system against the defined requirements or user stories to identify bugs manually, review the application and assist the development team in creating a defect-free software.

When do you need Manual Testing?

you want to do a quick smoke test then manual testing is best.

If you want to bring a thorough intervention to your application (web/Desktop/Mobile), then it can be achieved only through manual testing. It is inevitable in ad hoc or exploratory testing. Also, there are some workflows which require human intervention particularly.

If your project is small and simple then manual testing is the best choice. It removes the extra cost and time compared to the automation testing.

What can be done by a Manual Testing?

When something went wrong in the System Under Test (SUT) then manual testers can act swiftly to raise and retest the problem.

The software tester executes the test cases and generates the test reports manually.

What is Test Automation?

Test automation is a process of running test cases, managing test data, and generating the test results automatically.

When do you need a Test Automation?

If you have highly Repeatable, Determinant, Unopinionated test cases in your testing process then it's good to choose test automation.

Automated Testing is suited for Regression Testing, Performance Testing, Load Testing.

What are the uses of Test Automation?

  • Reduces your development cycles
  • Avoids inconvenient repetitive tasks
  • Improves software quality
  • Provides the maximum return on investment (ROI)
